Overview: How It Works

pqwave watches a .sch schematic file. When you save in lepton-schematic, pqwave exports a SPICE netlist, runs ngspice, and loads the results. A TCP server running inside lepton-schematic enables bidirectional cross-probe and value back-annotation. Small Scheme scripts installed once add in-schematic menus — Netlist > 2 SPICE, Simulation > ngspice, Wave > pqwave — following the same pattern as lepton-eda’s built-in allegro backend.

lepton-schematic                         pqwave
───────────────                          ──────
User edits .sch                         File watcher detects save
Saves (Ctrl+S) ─────────────────────►    lepton-netlist exports netlist
                                        ngspice -b -r result.raw
TCP server (port 9424) ◄─────────────►  Loads & displays all signals
  ← click net → plot trace               Cross-probe: highlight nets
  ← cursor move → DC stamp               Back-annotate: stamp trace values
No lepton-eda source modifications required. Everything works with a stock lepton-eda installation (1.9.18+). The integration uses lepton-eda’s own Guile Scheme extension mechanism — the same one used by its built-in netlist backends.

Getting Started

Prerequisites

  • lepton-eda 1.9.18+ (lepton-netlist and lepton-schematic in PATH)
  • ngspice 42+ in PATH
  • pqwave installed

Installing the Bridge Files

The bridge needs two Scheme files installed into lepton-eda’s configuration. Both are loaded via (load ...) lines appended to lepton-eda’s installed menu.scm. This is a one-time step. No source modifications, no rebuild required.

What gets installed:

~/.config/lepton-eda/pqwave-menus.scm    ← in-schematic menus (Netlist / Simulation / Wave)
~/.config/lepton-eda/pqwave-server.scm   ← TCP server for cross-probe & back-annotation
$LEPTON_PREFIX/share/lepton-eda/scheme/conf/schematic/menu.scm
    ↑ two (load ...) lines appended

Automated (GUI)

Open pqwave, then File → Lepton Bridge → Watch Schematic. Select any .sch file. A dialog will ask whether to install. Click Yes. Restart lepton-schematic.

When the bundled files are newer than the installed ones (after a pqwave upgrade), pqwave detects this automatically and offers to update.

Automated (Headless)

api.lepton_config("install_server")

Manual Install

Copy both files and append the load lines:

# 1. Copy both Scheme files to your config directory
cp pqwave/bridge/lepton/menu-additions.scm \
   ~/.config/lepton-eda/pqwave-menus.scm
cp pqwave/bridge/lepton/pqwave-server.scm \
   ~/.config/lepton-eda/pqwave-server.scm

# 2. Append load lines to lepton-eda's installed menu.scm
echo '(load "~/.config/lepton-eda/pqwave-menus.scm")' \
  >> $LEPTON_PREFIX/share/lepton-eda/scheme/conf/schematic/menu.scm
echo '(load "~/.config/lepton-eda/pqwave-server.scm")' \
  >> $LEPTON_PREFIX/share/lepton-eda/scheme/conf/schematic/menu.scm

Replace $LEPTON_PREFIX with your lepton-eda install path (e.g. /usr, /usr/local, ~/Apps/lepton-eda).

Verifying the Installation

Restart lepton-schematic. You should see three new menus:

  • Netlist → 2 SPICE
  • Simulation → ngspice
  • Wave → pqwave

Verify the TCP server is listening:

ss -tlnp | grep 9424
# Should show guile-3.0 listening on 127.0.0.1:9424

Netlist Pipeline

The bridge runs a three-stage pipeline on every save:

StageCommandOutput
Export lepton-netlist -g spice-sdb -o <file>.cir <file>.sch SPICE netlist
Simulate ngspice -b -r <file>.raw <file>.cir Raw simulation data
Load pqwave opens the .raw file Waveforms displayed

Unlike the KiCad bridge, lepton-eda’s spice-sdb backend produces clean SPICE. Zero post-processing fixes are needed — no slash stripping, no pin reordering, no control-block relocation.

SettingDefaultDescription
auto_simulatetrueRun simulation automatically on file save
port9424TCP port for cross-probe server

Set via lepton_config(key, value) or the Settings dialog (lepton-netlist and ngspice paths).

In-Schematic Menus

Once installed, lepton-schematic gains three menu entries between Netlist and Help, following the same pattern as the built-in “1 allegro” entry:

MenuItemWhat It Does
Netlist 2 SPICE Runs lepton-netlist -g spice-sdb on the current page, writes <basename>.cir in the schematic directory.
Simulation ngspice Generates the netlist, then runs ngspice -b -r <basename>.raw.
Wave pqwave Launches pqwave <basename>.raw in the background (does not block lepton-schematic).

The menu bar layout:

  [Options] [Netlist] [Simulation] [Wave] [Help]
                      1 allegro
                      2 SPICE

The typical workflow mirrors xschem’s pattern:

1. Netlist → 2 SPICE    →  circuit.cir
2. Simulation → ngspice  →  circuit.raw
3. Wave → pqwave          →  pqwave opens, plots traces

Cross-Probe & Back-Annotation

A TCP server runs inside lepton-schematic on port 9424, enabling three interaction modes between pqwave and the schematic:

Forward Cross-Probe (pqwave → schematic)

Right-click a trace legend and select Probe in Lepton-EDA. The corresponding net is highlighted and zoomed-to in the schematic. Multiple selected traces are all probed.

Cursor-driven: When you move the XA cursor along a trace, pqwave automatically highlights the corresponding net in the schematic (250 ms debounce). Works for all plotted traces simultaneously.

Reverse Cross-Probe (schematic → pqwave)

Click a net in lepton-schematic. pqwave receives the selected net name and automatically plots the corresponding v(netname) trace.

Value Back-Annotation (cursor → schematic)

As you move the XA cursor, the trace value at the cursor position is stamped onto the net’s name in the schematic as netname [DC:value V]. This works for all plotted traces simultaneously — each net shows its own value at the cursor position.

The value is the same one shown in the trace legend (computed via np.interp at the cursor X position).

DC Annotation

The control bar’s Annotate DC button stamps the mean voltage of every v(node) variable onto the schematic nets. Clear removes all annotations.

TCP Protocol

All cross-probe commands are plain text over TCP on port 9424:

CommandDirectionEffect
$NET: "name"pqwave → leptonHighlight and zoom to net
$CLEARpqwave → leptonClear selection
$ANNOTATE:DC name valuepqwave → leptonStamp DC voltage on netname
$CLEAR:DCpqwave → leptonRemove all DC stamps
$SELECTED:net namelepton → pqwaveUser clicked a net in schematic

Worked Tutorials

Every tutorial uses docs/lepton/examples/TwoStageAmp.sch. Open it in lepton-schematic to follow along. Ensure the bridge files are installed and lepton-schematic has been restarted.

Tutorial 1: TwoStageAmp — Complete Workflow

A two-stage BJT amplifier (2N3904 NPN transistors, 15 V supply, 1 kHz input). This walks through the full in-schematic workflow with cross-probe.

Circuit: Q1 + Q2 common-emitter stages, capacitive coupling, resistive biasing. The schematic includes a SPICE model include (./models/2N3904.mod) and simulation directives.

Step 1: Generate the Netlist

In lepton-schematic, click Netlist → 2 SPICE. This creates TwoStageAmp.cir in the same directory. Check the log window for confirmation.

Step 2: Run the Simulation

Click Simulation → ngspice. This generates the netlist (same as Step 1) and runs ngspice, producing TwoStageAmp.raw. The log window shows simulation progress.

Step 3: View Results

Click Wave → pqwave. pqwave launches with the .raw file loaded. You should see variables including v(vin), v(vbase1), v(vcoll1), v(vout), and time.

Step 4: Browse and Cross-Probe

Double-click v(vin) and v(vout) in the signal browser. v(vin) shows a 1 mV, 1 kHz sine. v(vout) shows the amplified output — approximately 240× gain (47.6 dB) with clipping at the 15 V supply rail.

Right-click a trace and choose Probe in Lepton-EDA to highlight the net in the schematic. Click a net in lepton-schematic to automatically plot its trace in pqwave. Move the cursor along a trace to see the live value stamped on the net name in the schematic.

Alternative: File-Watching Workflow

Instead of the in-schematic menus, let pqwave drive:

1. In pqwave: File → Lepton Bridge → Watch Schematic
2. Select TwoStageAmp.sch
3. In lepton-schematic: make a change, press Ctrl+S
4. pqwave auto-detects the save, re-simulates, and reloads
5. Cross-probe is active in both directions

Tutorial 2: Standalone Scripting

All bridge operations are available through the Session API for headless use:

import pqwave.session as session

api = session.SessionAPI()

# Install the bridge files (one-time)
api.lepton_config("install_server")

# Watch and simulate
api.lepton_watch("/home/user/projects/amp/TwoStageAmp.sch")
result = api.lepton_simulate()

print(f"Loaded {len(result['netlist'].splitlines())} netlist lines")
print(f"Raw file: {result['raw_file']}")

# Cross-probe
api.lepton_probe_net("Vout")

# Back-annotate DC operating point
api.lepton_annotate_dc({"Vbase1": 1.82, "Vout": 5.67})

# Clean up
api.lepton_clear_annotations()
api.lepton_unwatch()

Troubleshooting

lepton-netlist not found

Install lepton-eda or set the path in Settings → External Converter Paths → lepton-netlist. Verify with which lepton-netlist.

ngspice simulation errors

Check the chat panel for ngspice stderr output. Common causes: missing model files (the schematic references ./models/2N3904.mod — ensure it exists relative to the .sch file), convergence failures, or syntax errors in SPICE directives.

Menus don’t appear in lepton-schematic

  1. Verify both bridge files exist: ls ~/.config/lepton-eda/pqwave-menus.scm ~/.config/lepton-eda/pqwave-server.scm
  2. Verify the load lines are in menu.scm: grep pqwave $(lepton-cli shell -c "(display (lookup-sys-data-path \"scheme/conf/schematic/menu.scm\"))(force-output)" 2>/dev/null)
  3. Restart lepton-schematic after installing.
  4. Check the Log Window (View → Log Window) for Scheme errors.

Cross-probe not working

  1. Verify lepton-schematic is running and the TCP server is listening: ss -tlnp | grep 9424
  2. If not listening, check for stale Guile cache: rm -rf ~/.cache/guile/ccache/ and restart lepton-schematic.
  3. In pqwave, unwatch and re-watch to reconnect the TCP client.
  4. Check the chat panel for connection status messages.

Value back-annotation doesn’t update

Ensure the XA cursor is visible and placed on a trace. The value is computed from the same np.interp result shown in the trace legend. If the cursor is outside the trace’s X range, no value is sent.

File watcher doesn’t detect saves

The watcher polls mtime and file size every second. If you save twice within the same second, the second save may be missed. This is a known limitation of mtime polling. Workaround: wait >1 second between saves.

Port conflict

If port 9424 is already in use, change it with lepton_config("port", <new_port>) before starting the bridge. Restart lepton-schematic after changing the port in the server file.

Uninstalling

Remove the (load ...) lines from menu.scm and delete ~/.config/lepton-eda/pqwave-menus.scm and ~/.config/lepton-eda/pqwave-server.scm. Restart lepton-schematic.
