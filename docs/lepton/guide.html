Overview: How It Works

pqwave watches a .sch schematic file. When you save in lepton-schematic, pqwave exports a SPICE netlist, runs ngspice, and loads the results. A small Scheme script installed once adds in-schematic menus — Netlist > SPICE, Simulation > ngspice, Wave View > pqwave — following the same pattern as lepton-eda’s built-in allegro backend, without modifying a single line of C code.

lepton-schematic                         pqwave
───────────────                          ──────
User edits .sch                         File watcher detects save
Saves (Ctrl+S) ─────────────────────►    lepton-netlist exports netlist
                                        ngspice -b -r result.raw
                                        Loads & displays all signals
                                        ──────────────────────────
                                        Cross-probe: highlight nets
                                        Back-annotate: stamp DC voltages
No lepton-eda source modifications required. Everything works with a stock lepton-eda installation (1.9.18+). The integration uses lepton-eda’s own Guile Scheme extension mechanism — the same one used by its built-in netlist backends.

Getting Started

Prerequisites

  • lepton-eda 1.9.18+ (lepton-netlist and lepton-schematic in PATH)
  • ngspice 42+ in PATH
  • pqwave installed

Installing the Menu Additions

The bridge adds in-schematic menus (Netlist → SPICE, Simulation → ngspice, Wave View → pqwave) by appending a single (load ...) line to lepton-eda’s installed menu.scm. This is a one-time step. No source modifications, no rebuild required.

What gets installed:

~/.config/lepton-eda/pqwave-menus.scm   ← the menu script (47 lines)
$LEPTON_PREFIX/share/lepton-eda/scheme/conf/schematic/menu.scm
    ↑ one (load ...) line appended

Automated (GUI)

Open pqwave, then File → Lepton Bridge → Watch Schematic. Select any .sch file. A dialog will ask whether to install. Click Yes. Restart lepton-schematic.

Automated (Headless)

api.lepton_config("install_server")

Manual Install

You can also install by hand — two steps:

# 1. Copy the menu additions to your config directory
cp pqwave/bridge/lepton/menu-additions.scm \
   ~/.config/lepton-eda/pqwave-menus.scm

# 2. Append the load line to lepton-eda's installed menu.scm
echo '(load "~/.config/lepton-eda/pqwave-menus.scm")' \
  >> $LEPTON_PREFIX/share/lepton-eda/scheme/conf/schematic/menu.scm

Replace $LEPTON_PREFIX with your lepton-eda install path (e.g. /usr, /usr/local, ~/Apps/lepton-eda).

Verifying the Installation

Restart lepton-schematic. You should see three new menus:

  • Netlist → SPICE
  • Simulation → ngspice
  • Wave View → pqwave

pqwave also checks installation status when you start watching a file. If the bundled version is newer than the installed one (after upgrading pqwave), it offers to update.

Netlist Pipeline

The bridge runs a three-stage pipeline on every save:

StageCommandOutput
Export lepton-netlist -g spice-sdb -o <file>.cir <file>.sch SPICE netlist
Simulate ngspice -b -r <file>.raw <file>.cir Raw simulation data
Load pqwave opens the .raw file Waveforms displayed

Unlike the KiCad bridge, lepton-eda’s spice-sdb backend produces clean SPICE. Zero post-processing fixes are needed — no slash stripping, no pin reordering, no control-block relocation.

SettingDefaultDescription
auto_simulatetrueRun simulation automatically on file save
port9424TCP port for cross-probe server

Set via lepton_config(key, value) or the Settings dialog (lepton-netlist and ngspice paths).

In-Schematic Menus

Once pqwave-server.scm is loaded, lepton-schematic gains three menu entries for driving the simulation workflow without leaving the editor:

MenuActionWhat It Does
Netlist → SPICE &spice-netlist Runs lepton-netlist -g spice-sdb on the current page, writes <basename>.cir in the schematic directory.
Simulation → ngspice &sim-ngspice Generates the netlist, then runs ngspice -b -r <basename>.raw.
Wave View → pqwave &wave-pqwave Launches pqwave <basename>.raw.

The typical workflow mirrors xschem’s pattern:

1. Netlist → SPICE       →  circuit.cir
2. Simulation → ngspice  →  circuit.raw
3. Wave View → pqwave    →  pqwave opens, plots traces

Bidirectional Cross-Probe

Cross-probe works in both directions over a single TCP connection on port 9424. pqwave is the client; lepton-schematic (running the Scheme script) is the server.

pqwave → Schematic

Select a trace in pqwave, then File → Lepton Bridge → Cross-Probe → Probe Selected Net. The corresponding net is highlighted in lepton-schematic and the view centers on it.

Under the hood, pqwave sends:

$NET: "Vbase1"

The Scheme server calls select-object! on every net segment with netname=Vbase1, then calls schematic_canvas_zoom_object to center the view.

Schematic → pqwave

Click a net in lepton-schematic. pqwave automatically plots the corresponding waveform trace (if it exists in the loaded dataset).

Under the hood, lepton-schematic’s select-objects-hook fires on every user click. The Scheme server extracts the netname= attribute and sends:

$SELECTED:net Vbase1

pqwave receives this, searches the loaded dataset for v(Vbase1) or Vbase1, and plots the trace.

No feedback loop. When pqwave sends $NET: and the server calls select-object!, the hook does not fire. The hook is only triggered by user clicks in the C code (o_attrib_add_selected), not by programmatic select-object! calls. The two directions use distinct mechanisms, preventing infinite ping-pong.

Back-Annotation

Simulation results can be written back onto the schematic using two mechanisms:

DC Bias Stamps

After a simulation completes, click Annotate DC in the Lepton control bar (or File → Lepton Bridge → Annotate DC). pqwave extracts the mean voltage of every v(node) variable and stamps it onto the corresponding netname attribute:

BeforeAfter
netname=Vbase1netname=Vbase1 [DC:1.82 V]
netname=Voutnetname=Vout [DC:5.67 V]

These stamps persist in the .sch file. To remove them, click Clear Annotations (restores original netname values by stripping the [DC:...V] suffix).

Floating Labels

The Scheme server can also create text objects at arbitrary coordinates on the schematic page (used via the API for custom annotation layers):

bridge.annotate_label("Vbase1", "1.82 V", 30300, 49850)

These are tracked by the server and can be bulk-removed without affecting DC stamps.

Worked Tutorials

Every tutorial uses docs/lepton/examples/TwoStageAmp.sch. Open it in lepton-schematic to follow along. Ensure pqwave-server.scm is installed and lepton-schematic has been restarted.

Tutorial 1: TwoStageAmp — Complete Workflow

A two-stage BJT amplifier (2N3904 NPN transistors, 15 V supply, 1 kHz input). This walks through the full in-schematic workflow.

Circuit: Q1 + Q2 common-emitter stages, capacitive coupling, resistive biasing. The schematic includes a SPICE model include (./models/2N3904.mod) and simulation directives.

Step 1: Generate the Netlist

In lepton-schematic, click Netlist → SPICE. This creates TwoStageAmp.cir in the same directory. Check the log window for confirmation.

Step 2: Run the Simulation

Click Simulation → ngspice. This generates the netlist (same as Step 1) and runs ngspice, producing TwoStageAmp.raw. The log window shows simulation progress.

Step 3: View Results

Click Wave View → pqwave. pqwave launches with the .raw file loaded. You should see variables including v(vin), v(vbase1), v(vcoll1), v(vout), and time.

Step 4: Browse Signals

Double-click v(vin) and v(vout) in the signal browser. v(vin) shows a 1 mV, 1 kHz sine. v(vout) shows the amplified output — approximately 240× gain (47.6 dB) with clipping at the 15 V supply rail.

Alternative: File-Watching Workflow

Instead of the in-schematic menus, let pqwave drive:

1. In pqwave: File → Lepton Bridge → Watch Schematic
2. Select TwoStageAmp.sch
3. In lepton-schematic: make a change, press Ctrl+S
4. pqwave auto-detects the save, re-simulates, and reloads

Tutorial 2: Bidirectional Cross-Probe

pqwave → Schematic

  1. After simulation, select the v(vout) trace in pqwave.
  2. File → Lepton Bridge → Cross-Probe → Probe Selected Net.
  3. lepton-schematic highlights all net segments with netname=Vout and centers the view on them.

Schematic → pqwave

  1. In lepton-schematic, click any net segment (e.g., the net labeled Vbase1).
  2. pqwave’s chat panel shows: [lepton] Selected net: Vbase1.
  3. The v(vbase1) trace is automatically plotted.
  4. Click another net (e.g., Vcoll1) — that trace is plotted too.

Clicking a net that is already plotted does nothing (duplicate detection).

Tutorial 3: DC Back-Annotation

Step 1: Stamp DC Voltages

After simulation completes, click Annotate DC in the Lepton control bar. The chat panel confirms:

[lepton] DC annotations stamped: 8 nets

In lepton-schematic, the netname attributes now include DC values:

netname=Vbase1 [DC:1.82 V]
netname=Vcoll1 [DC:8.23 V]
netname=Vout [DC:5.67 V]
...

Step 2: Verify

Compare the stamped values against cursor measurements in pqwave:

  1. Place cursor A on the v(vbase1) trace.
  2. The status bar shows the Y value — it should match the stamped DC voltage (~1.82 V for the bias point).

Step 3: Clear and Restore

Click Clear Annotations. The DC stamps are removed and the netname attributes are restored to their original values. Floating labels (if any) are also cleared.

DC stamps persist in the .sch file. If you save the schematic with annotations active, the stamps become part of the file. Clear annotations before saving unless you want them permanently.

Tutorial 4: Standalone Scripting

All bridge operations are available through the Session API for headless use:

import pqwave.session as session

api = session.SessionAPI()

# Install the Scheme server (one-time)
api.lepton_config("install_server")

# Watch and simulate
api.lepton_watch("/home/user/projects/amp/TwoStageAmp.sch")
result = api.lepton_simulate()

print(f"Loaded {len(result['netlist'].splitlines())} netlist lines")
print(f"Raw file: {result['raw_file']}")

# Cross-probe
api.lepton_probe_net("Vout")

# Back-annotate
api.lepton_annotate_dc({"Vbase1": 1.82, "Vout": 5.67})

# Clean up
api.lepton_clear_annotations()
api.lepton_unwatch()

Troubleshooting

“lepton-netlist not found”

Install lepton-eda or set the path in Settings → External Converter Paths → lepton-netlist. Verify with which lepton-netlist.

ngspice simulation errors

Check the chat panel for ngspice stderr output. Common causes: missing model files (the schematic references ./models/2N3904.mod — ensure it exists relative to the .sch file), convergence failures, or syntax errors in SPICE directives.

Menus don’t appear in lepton-schematic

  1. Verify the menu additions file exists: ls ~/.config/lepton-eda/pqwave-menus.scm
  2. Verify the load line is in menu.scm: grep pqwave-menus $(dirname $(which lepton-schematic))/../share/lepton-eda/scheme/conf/schematic/menu.scm
  3. Restart lepton-schematic after installing.

Cross-probe (future feature)

Cross-probe and back-annotation require a TCP server running inside lepton-schematic — this is not yet implemented. The in-schematic menus (Netlist → SPICE, Simulation → ngspice, Wave View → pqwave) are the currently available features.

File watcher doesn’t detect saves

The watcher polls mtime and file size every second. If you save twice within the same second, the second save may be missed. This is a known limitation of mtime polling. Workaround: wait >1 second between saves.

Uninstalling

Remove the (load ...) line from menu.scm and delete ~/.config/lepton-eda/pqwave-menus.scm. Restart lepton-schematic.
