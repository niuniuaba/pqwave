Qucs-S Bridge

pqwave can act as a waveform viewer for Qucs-S simulations. Once configured, every simulation you run in Qucs-S automatically opens its results in pqwave — with traceable output files in a simulation/ subdirectory next to your schematic.

How it works: pqwave replaces ngspice as the simulator executable in Qucs-S's settings. When Qucs-S clicks "Simulate," pqwave runs the real ngspice, copies the output files, and opens them for viewing.

Setup

  1. Close Qucs-S if it is running.
  2. In pqwave: File → Qucs-S Bridge → Connect
  3. Click OK in the confirmation dialog.
  4. Start Qucs-S.

That's it. Now simulate any schematic in Qucs-S — pqwave opens the results automatically.

⚠ Important: Qucs-S loads simulator settings only at startup. You must close Qucs-S before Connect or Disconnect, and start it after the change.

After Setup

When you click Simulate in Qucs-S:

  1. Qucs-S generates the SPICE netlist as usual
  2. pqwave runs ngspice in batch mode
  3. Output files are copied to a simulation/ directory next to your schematic:
~/projects/bridge/
├── bridge.sch
└── simulation/
    ├── bridge.cir          ← netlist
    ├── bridge.tr1.plot     ← transient simulation
    └── bridge.ac1.raw      ← AC simulation

Qucs-S's own diagram system continues to work — the integration is additive.

Disconnecting

  1. Close Qucs-S if it is running.
  2. In pqwave: File → Qucs-S Bridge → Disconnect
  3. Click OK to restore the original configuration.

Qucs-S will return to using ngspice directly.

Manual Setup

If the one-click setup doesn't work, configure manually:

  1. In Qucs-S: Simulation → Simulator settings
  2. Set Ngspice executable location to the path of pqwave:
/home/<user>/.local/bin/pqwave
  1. Set Ngspice CLI parameters to:
--qucs-bridge
  1. Click Apply changes
  2. Restart Qucs-S

Troubleshooting

pqwave doesn't open after simulation

  • Did you restart Qucs-S after Connect? Qucs-S loads settings only at startup.
  • Check that pqwave is on your PATH: which pqwave
  • Test the bridge manually: pqwave --qucs-bridge spice4qucs.cir
  • Verify ngspice is installed: which ngspice

pqwave opens but shows "Error opening file"

  • The config may be in a broken half-state. Run Connect again to repair it.
  • Check that NgspiceParams is set to --qucs-bridge (not empty and not inside NgspiceExecutable).

Changes don't take effect

  • Qucs-S caches NgspiceExecutable at startup. Close Qucs-S before Connect or Disconnect.
  • Qucs-S overwrites the config on exit with its cached values. If you changed settings while Qucs-S was running, they may be lost.

Undoing the integration

  • Use File → Qucs-S Bridge → Disconnect in pqwave
  • Or restore from backup: cp ~/.config/qucs/qucs_s.conf.pqwave_backup ~/.config/qucs/qucs_s.conf
  • Or in Qucs-S, manually set Ngspice executable back to /usr/bin/ngspice

Environment Variables

VariableEffect
PQWAVE_NGSPICE Override path to ngspice binary (e.g. /opt/local/bin/ngspice)
PQWAVE_QUCS_AUTO_OPEN=0 Disable automatic pqwave launch after simulation (outputs still copied)

CLI Reference

# Configure integration from command line
pqwave --setup-qucs-integration

# Start pqwave and auto-connect to Qucs-S bridge
pqwave --connect qucs

# Manual post-simulation hook (normally called by Qucs-S)
pqwave --qucs-bridge spice4qucs.cir
